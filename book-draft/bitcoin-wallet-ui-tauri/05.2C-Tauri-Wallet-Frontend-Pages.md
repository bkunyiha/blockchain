# Chapter 28C: Tauri Wallet — Frontend Pages (Complete Code Walkthrough)

This companion chapter contains the **complete, verbatim source** for all 7 page components in the wallet app. The pages follow three interaction patterns established in Chapter 26C (the admin app):

| Pattern | Description | Pages |
|---------|-------------|-------|
| **Auto-fetch** | Load data when an active wallet exists, using React Query's `useQuery` | WalletInfoPage, BalancePage, HistoryPage |
| **List + manage** | Fetch a collection and provide CRUD actions | WalletListPage |
| **Form submission** | User fills a form, validates, and triggers a mutation | CreateWalletPage, SendPage, SettingsPage |

---

## Listing 19C.1: `src/pages/wallet/CreateWalletPage.tsx`


> - Form submission pattern: label input → `useCreateWallet` mutation → navigate on success
> - Backend: `create_wallet` command → API call + database persist

```tsx
import React, { useState } from "react";
import { useNavigate } from "react-router-dom";
import { Loader } from "lucide-react";
import { useCreateWallet } from "../../hooks/useCommands";
import { useWalletStore } from "../../store/walletStore";
import { useToastStore } from "../../store/toastStore";
import { cn } from "../../lib/utils";

export default function CreateWalletPage() {
  const navigate = useNavigate();
  const [label, setLabel] = useState("");
  const { mutate: createWallet, isPending } = useCreateWallet();
  const { setActiveWallet } = useWalletStore();
  const theme = useWalletStore((s) => s.theme);
  const addToast = useToastStore((s) => s.addToast);

  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault();
    createWallet(label || undefined, {
      onSuccess: (wallet) => {
        setActiveWallet(wallet);
        addToast("success", `Wallet created: ${wallet.label || "Unnamed"}`);
        navigate("/wallet/list");
      },
      onError: (error: any) => {
        addToast("error", error.message || "Failed to create wallet");
      },
    });
  };

  return (
    <div className="max-w-2xl mx-auto">
      <div className="mb-6">
        <h1 className="text-3xl font-bold mb-2">Create New Wallet</h1>
        <p className={cn("text-sm", theme === "light" ? "text-gray-600" : "text-gray-400")}>
          Create a new Bitcoin wallet to get started
        </p>
      </div>

      <form onSubmit={handleSubmit} className={cn(
        "rounded-lg border p-6",
        theme === "light" ? "bg-white border-gray-200" : "bg-gray-800 border-gray-700"
      )}>
        <div className="mb-4">
          <label className="block text-sm font-medium mb-2">Wallet Label (Optional)</label>
          <input
            type="text"
            value={label}
            onChange={(e) => setLabel(e.target.value)}
            placeholder="e.g., My Main Wallet"
            className={cn(
              "w-full px-15 py-2 rounded-lg border transition-colors",
              theme === "light"
                ? "bg-white border-gray-300 focus:border-blue-500 focus:ring-1 focus:ring-blue-500"
                : "bg-gray-700 border-gray-600 focus:border-blue-500 focus:ring-1 focus:ring-blue-500"
            )}
          />
        </div>

        <div className={cn(
          "p-4 rounded-lg mb-20 text-sm",
          theme === "light" ? "bg-blue-50 border border-blue-200 text-blue-700" : "bg-blue-900/20 border border-blue-800/50 text-blue-300"
        )}>
          A new Bitcoin address will be generated and securely stored locally.
        </div>

        <button
          type="submit"
          disabled={isPending}
          className="w-full bg-bitcoin-orange text-white py-2 rounded-lg font-medium hover:bg-bitcoin-orange-dark transition-colors disabled:opacity-50 disabled:cursor-not-allowed flex items-center justify-center gap-2"
        >
          {isPending && <Loader size={16} className="animate-spin" />}
          {isPending ? "Creating..." : "Create Wallet"}
        </button>
      </form>
    </div>
  );
}
```

This is one of the simplest pages in the app. The flow is straightforward: the user enters an optional label, clicks "Create Wallet," and the `useCreateWallet` mutation fires. On success, three things happen in sequence: the new wallet is set as active (Zustand), a success toast appears, and the user is redirected to the wallet list.

The `label || undefined` pattern ensures we send `null` to the Rust backend when the label is empty (since `undefined` becomes `null` through Tauri's JSON serialization), rather than an empty string.

## Listing 19C.2: `src/pages/wallet/WalletListPage.tsx`


> - List + manage pattern: `useSavedWallets` query + `useDeleteWallet` mutation + inline `commands.setActiveWallet` and `commands.updateWalletLabel`
> - Backend: `get_saved_wallets`, `set_active_wallet`, `delete_wallet_address`, `update_wallet_label`

```tsx
import React from "react";
import { Loader, AlertCircle } from "lucide-react";
import { useSavedWallets, useDeleteWallet, commands } from "../../hooks/useCommands";
import { useWalletStore } from "../../store/walletStore";
import { useToastStore } from "../../store/toastStore";
import WalletCard from "../../components/WalletCard";
import { cn } from "../../lib/utils";

export default function WalletListPage() {
  const { data: wallets, isLoading, error } = useSavedWallets();
  const { mutate: deleteWallet } = useDeleteWallet();
  const { activeWallet, setActiveWallet } = useWalletStore();
  const theme = useWalletStore((s) => s.theme);
  const addToast = useToastStore((s) => s.addToast);

  const handleSelectWallet = async (address: string) => {
    try {
      await commands.setActiveWallet(address);
      const selectedWallet = wallets?.find((w) => w.address === address);
      if (selectedWallet) {
        setActiveWallet(selectedWallet);
        addToast("success", `Wallet selected: ${selectedWallet.label || "Unnamed"}`);
      }
    } catch (err: any) {
      addToast("error", err.message || "Failed to select wallet");
    }
  };

  const handleDeleteWallet = (address: string) => {
    deleteWallet(address, {
      onSuccess: () => {
        if (activeWallet?.address === address) {
          setActiveWallet(null);
        }
        addToast("success", "Wallet deleted");
      },
      onError: (err: any) => {
        addToast("error", err.message || "Failed to delete wallet");
      },
    });
  };

  const handleRenameWallet = async (address: string, newLabel: string) => {
    try {
      await commands.updateWalletLabel(address, newLabel);
      if (activeWallet?.address === address) {
        setActiveWallet({ ...activeWallet, label: newLabel || null });
      }
      addToast("success", "Wallet renamed");
    } catch (err: any) {
      addToast("error", err.message || "Failed to rename wallet");
    }
  };

  if (isLoading) {
    return (
      <div className="flex flex-col items-center justify-center h-full">
        <Loader size={32} className="animate-spin text-bitcoin-orange mb-4" />
        <p className={cn("text-sm", theme === "light" ? "text-gray-600" : "text-gray-400")}>
          Loading wallets...
        </p>
      </div>
    );
  }

  if (error) {
    return (
      <div className="max-w-2xl mx-auto">
        <div className={cn(
          "flex items-start gap-15 p-4 rounded-lg border",
          theme === "light" ? "bg-red-50 border-red-200" : "bg-red-900/20 border-red-800/50"
        )}>
          <AlertCircle size={20} className={cn(
            "flex-shrink-0 mt-0.5",
            theme === "light" ? "text-red-600" : "text-red-400"
          )} />
          <div>
            <h3 className={cn("font-semibold", theme === "light" ? "text-red-800" : "text-red-200")}>
              Error Loading Wallets
            </h3>
            <p className={cn("text-sm mt-1", theme === "light" ? "text-red-700" : "text-red-300")}>
              {error instanceof Error ? error.message : "An unexpected error occurred"}
            </p>
          </div>
        </div>
      </div>
    );
  }

  return (
    <div className="max-w-4xl mx-auto">
      <div className="mb-6">
        <h1 className="text-3xl font-bold mb-2">My Wallets</h1>
        <p className={cn("text-sm", theme === "light" ? "text-gray-600" : "text-gray-400")}>
          {wallets?.length || 0} wallet{wallets?.length !== 1 ? "s" : ""} found
        </p>
      </div>

      {!wallets || wallets.length === 0 ? (
        <div className={cn(
          "rounded-lg border p-22 text-center",
          theme === "light" ? "bg-white border-gray-200" : "bg-gray-800 border-gray-700"
        )}>
          <p className={cn("text-sm mb-4", theme === "light" ? "text-gray-600" : "text-gray-400")}>
            No wallets yet. Create your first wallet to get started.
          </p>
        </div>
      ) : (
        <div className="grid gap-4">
          {wallets.map((wallet) => (
            <WalletCard
              key={wallet.address}
              wallet={wallet}
              isActive={activeWallet?.address === wallet.address}
              onSelect={() => handleSelectWallet(wallet.address)}
              onDelete={() => handleDeleteWallet(wallet.address)}
              onRename={(label) => handleRenameWallet(wallet.address, label)}
            />
          ))}
        </div>
      )}
    </div>
  );
}
```

The wallet list page is the hub of the application. It displays all saved wallets as `WalletCard` components and handles three management operations:

**Select**: Calls the backend's `set_active_wallet` to validate and persist the selection, then updates the Zustand store. This two-step approach ensures the backend and frontend agree on which wallet is active.

**Delete**: Uses the `useDeleteWallet` mutation (which invalidates the `["savedWallets"]` query). Additionally checks if the deleted wallet was the active one, and clears the selection if so.

**Rename**: Calls `commands.updateWalletLabel` directly (not through a mutation hook). If the renamed wallet is the active one, the Zustand store is updated inline with the spread operator to preserve reactivity.

## Listing 19C.3: `src/pages/wallet/WalletInfoPage.tsx`


> - Auto-fetch pattern: `useWalletInfo(activeWallet?.address)` triggers when a wallet is active
> - Backend: `get_wallet_info` command → API call

```tsx
import React from "react";
import { Loader, AlertCircle, Copy, Check } from "lucide-react";
import { useWalletInfo } from "../../hooks/useCommands";
import { useWalletStore } from "../../store/walletStore";
import { useToastStore } from "../../store/toastStore";
import { truncateAddress, cn } from "../../lib/utils";
import JsonViewer from "../../components/JsonViewer";
import { writeText } from "@tauri-apps/plugin-clipboard-manager";

export default function WalletInfoPage() {
  const { activeWallet } = useWalletStore();
  const { data: walletInfo, isLoading, error } = useWalletInfo(activeWallet?.address);
  const theme = useWalletStore((s) => s.theme);
  const addToast = useToastStore((s) => s.addToast);
  const [copied, setCopied] = React.useState(false);

  const handleCopyAddress = async () => {
    if (activeWallet) {
      try {
        await writeText(activeWallet.address);
        setCopied(true);
        addToast("success", "Address copied to clipboard");
        setTimeout(() => setCopied(false), 2000);
      } catch {
        addToast("error", "Failed to copy address");
      }
    }
  };

  if (isLoading) {
    return (
      <div className="flex flex-col items-center justify-center h-full">
        <Loader size={32} className="animate-spin text-bitcoin-orange mb-4" />
        <p className={cn("text-sm", theme === "light" ? "text-gray-600" : "text-gray-400")}>
          Loading wallet info...
        </p>
      </div>
    );
  }

  if (error) {
    return (
      <div className="max-w-2xl mx-auto">
        <div className={cn(
          "flex items-start gap-15 p-4 rounded-lg border",
          theme === "light" ? "bg-red-50 border-red-200" : "bg-red-900/20 border-red-800/50"
        )}>
          <AlertCircle size={20} className={cn(
            "flex-shrink-0 mt-0.5",
            theme === "light" ? "text-red-600" : "text-red-400"
          )} />
          <div>
            <h3 className={cn("font-semibold", theme === "light" ? "text-red-800" : "text-red-200")}>
              Error Loading Wallet Info
            </h3>
            <p className={cn("text-sm mt-1", theme === "light" ? "text-red-700" : "text-red-300")}>
              {error instanceof Error ? error.message : "An unexpected error occurred"}
            </p>
          </div>
        </div>
      </div>
    );
  }

  return (
    <div className="max-w-4xl mx-auto">
      <div className="mb-6">
        <h1 className="text-3xl font-bold mb-2">Wallet Information</h1>
        <p className={cn("text-sm", theme === "light" ? "text-gray-600" : "text-gray-400")}>
          Details for {activeWallet?.label || "unnamed wallet"}
        </p>
      </div>

      {/* Address Card */}
      <div className={cn(
        "rounded-lg border p-20 mb-6",
        theme === "light" ? "bg-white border-gray-200" : "bg-gray-800 border-gray-700"
      )}>
        <h2 className="font-semibold mb-3">Address</h2>
        <div className="flex items-center gap-2">
          <code className={cn(
            "flex-1 p-15 rounded-lg border font-mono text-sm break-all",
            theme === "light"
              ? "bg-gray-50 border-gray-300 text-gray-800"
              : "bg-gray-700 border-gray-600 text-gray-200"
          )}>
            {activeWallet?.address}
          </code>
          <button
            onClick={handleCopyAddress}
            className="p-2 hover:text-bitcoin-orange transition-colors flex-shrink-0"
            title="Copy address"
          >
            {copied ? <Check size={18} className="text-green-400" /> : <Copy size={18} />}
          </button>
        </div>
      </div>

      {/* Metadata */}
      <div className={cn(
        "rounded-lg border p-20 mb-20 grid grid-cols-2 gap-4",
        theme === "light" ? "bg-white border-gray-200" : "bg-gray-800 border-gray-700"
      )}>
        <div>
          <p className={cn("text-xs font-medium mb-1", theme === "light" ? "text-gray-500" : "text-gray-400")}>
            Label
          </p>
          <p className="text-sm font-semibold">{activeWallet?.label || "Unnamed"}</p>
        </div>
        <div>
          <p className={cn("text-xs font-medium mb-1", theme === "light" ? "text-gray-500" : "text-gray-400")}>
            Created
          </p>
          <p className="text-sm">{new Date(activeWallet?.created_at || "").toLocaleDateString()}</p>
        </div>
      </div>

      {/* Full Response */}
      {walletInfo && <JsonViewer data={walletInfo} title="Wallet Info Response" />}
    </div>
  );
}
```

This page displays three sections: the full address (with copy), metadata (label and creation date from the local database), and the full API response via `JsonViewer`. The data comes from two sources: `activeWallet` from Zustand (local metadata) and `walletInfo` from React Query (remote API data).

## Listing 19C.4: `src/pages/wallet/BalancePage.tsx`


> - Auto-fetch pattern: `useBalance(activeWallet?.address)` → displays BTC + satoshis
> - Backend: `get_balance` command → API call

```tsx
import React from "react";
import { Loader, AlertCircle, RotateCw } from "lucide-react";
import { useBalance } from "../../hooks/useCommands";
import { useWalletStore } from "../../store/walletStore";
import { satoshisToBtc, cn } from "../../lib/utils";
import JsonViewer from "../../components/JsonViewer";
import { useQueryClient } from "@tanstack/react-query";

export default function BalancePage() {
  const { activeWallet } = useWalletStore();
  const { data: balanceData, isLoading, error, refetch } = useBalance(activeWallet?.address);
  const theme = useWalletStore((s) => s.theme);
  const queryClient = useQueryClient();

  const handleRefresh = () => {
    refetch();
  };

  if (isLoading) {
    return (
      <div className="flex flex-col items-center justify-center h-full">
        <Loader size={32} className="animate-spin text-bitcoin-orange mb-4" />
        <p className={cn("text-sm", theme === "light" ? "text-gray-600" : "text-gray-400")}>
          Fetching balance...
        </p>
      </div>
    );
  }

  if (error) {
    return (
      <div className="max-w-2xl mx-auto">
        <div className={cn(
          "flex items-start gap-15 p-4 rounded-lg border",
          theme === "light" ? "bg-red-50 border-red-200" : "bg-red-900/20 border-red-800/50"
        )}>
          <AlertCircle size={20} className={cn(
            "flex-shrink-0 mt-0.5",
            theme === "light" ? "text-red-600" : "text-red-400"
          )} />
          <div>
            <h3 className={cn("font-semibold", theme === "light" ? "text-red-800" : "text-red-200")}>
              Error Fetching Balance
            </h3>
            <p className={cn("text-sm mt-1", theme === "light" ? "text-red-700" : "text-red-300")}>
              {error instanceof Error ? error.message : "An unexpected error occurred"}
            </p>
          </div>
        </div>
      </div>
    );
  }

  const btcBalance = balanceData && typeof balanceData === "object" && "balance" in balanceData
    ? satoshisToBtc(balanceData.balance)
    : "0";

  return (
    <div className="max-w-4xl mx-auto">
      <div className="mb-20 flex items-center justify-between">
        <div>
          <h1 className="text-3xl font-bold mb-2">Wallet Balance</h1>
          <p className={cn("text-sm", theme === "light" ? "text-gray-600" : "text-gray-400")}>
            Current balance for your wallet
          </p>
        </div>
        <button
          onClick={handleRefresh}
          className="p-2 rounded-lg hover:bg-gray-700 transition-colors"
          title="Refresh balance"
        >
          <RotateCw size={20} />
        </button>
      </div>

      {/* Balance Card */}
      <div className={cn(
        "rounded-lg border p-22 mb-20 text-center",
        theme === "light" ? "bg-white border-gray-200" : "bg-gray-800 border-gray-700"
      )}>
        <p className={cn("text-sm font-medium mb-2", theme === "light" ? "text-gray-600" : "text-gray-400")}>
          Total Balance
        </p>
        <p className="text-5xl font-bold text-bitcoin-orange mb-2">{btcBalance} BTC</p>
        {balanceData && typeof balanceData === "object" && "balance" in balanceData && (
          <p className={cn("text-xs", theme === "light" ? "text-gray-500" : "text-gray-400")}>
            {balanceData.balance.toLocaleString()} satoshis
          </p>
        )}
      </div>

      {/* Full Response */}
      {balanceData && <JsonViewer data={balanceData} title="Balance Response" />}
    </div>
  );
}
```

The balance page centers on a large, prominent balance display in BTC (converted from satoshis using `satoshisToBtc`). The refresh button uses React Query's `refetch()` to manually re-fetch the balance from the API. The full response is available in the collapsible `JsonViewer` below.

The Send page is the most complex page in the wallet app. It implements a three-step flow: fill form → confirm in modal → execute transaction.

Key design decisions:

- The "From" address is read-only and comes from the active wallet — you can only send from the currently selected wallet
- The amount input accepts satoshis (integer), with a live BTC conversion displayed below
- Client-side validation runs before showing the modal (address not empty, amount > 0, wallet selected)
- The confirmation modal shows all transaction details (from, to, amount in both units) so the user can review before committing
- On success, the form is cleared and a success card appears with the TXID (copyable)
- The submit button uses `bg-green-600` (green) instead of the standard bitcoin orange to visually distinguish "send money" from other actions

```tsx
          <li>Keep your API key secure and never share it</li>
          <li>Use HTTPS URLs for secure communication</li>
          <li>Test your connection after updating settings</li>
        </ul>
      </div>
    </div>
  );
}
```

The Settings page demonstrates the **load → edit → save** pattern for persisted data. On mount, `useSettings()` loads the current settings from the database via the Rust backend. The `useEffect` populates the form fields when data arrives. When the user saves, `useSaveSettings` persists to the database and updates the in-memory state in one operation.

The connection test is independent of saving — you can test before or after saving. The test uses the current in-memory state (which may differ from what is in the form if the user has not saved yet).

The API key field uses `type="password"` to mask the value — a small but important UX detail for a wallet application.

## Summary: Page patterns at a glance

| Page | Pattern | Backend Commands | Key Hooks |
|------|---------|-----------------|-----------|
| CreateWalletPage | Form → mutation | `create_wallet` | `useCreateWallet` |
| WalletListPage | List + manage | `get_saved_wallets`, `set_active_wallet`, `delete_wallet_address`, `update_wallet_label` | `useSavedWallets`, `useDeleteWallet` |
| WalletInfoPage | Auto-fetch | `get_wallet_info` | `useWalletInfo` |
| BalancePage | Auto-fetch + refresh | `get_balance` | `useBalance` |
| SendPage | Form → confirm → mutation | `send_transaction` | `useSendTransaction` |
| HistoryPage | Auto-fetch + refresh | `get_tx_history` | `useTxHistory` |
| SettingsPage | Load → edit → save + test | `get_settings`, `save_settings`, `check_connection` | `useSettings`, `useSaveSettings` |

All pages share the same loading spinner, error display, and theme-aware styling patterns. The consistent structure makes it straightforward to add new wallet features by following the established patterns.
