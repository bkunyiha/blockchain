## Code Listing 17B-2 — Application Bootstrap (`main.tsx`)

This is the entry point. We create the React Query client, wrap the app in providers, and mount it to the DOM.

```tsx
import React from 'react';
import ReactDOM from 'react-dom/client';
import { BrowserRouter } from 'react-router-dom';
import { QueryClient, QueryClientProvider } from '@tanstack/react-query';
import App from './App';
import './index.css';

const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      staleTime: 1000 * 60 * 5, // 5 minutes
      gcTime: 1000 * 60 * 10, // 10 minutes
    },
  },
});

ReactDOM.createRoot(document.getElementById('root')!).render(
  <React.StrictMode>
    <QueryClientProvider client={queryClient}>
      <BrowserRouter>
        <App />
      </BrowserRouter>
    </QueryClientProvider>
  </React.StrictMode>
);
```

The `QueryClient` configuration sets two important defaults:

- **`staleTime: 5 minutes`** — After fetching data, React Query considers it "fresh" for 5 minutes. During this window, navigating back to a page shows cached data instantly without re-fetching. This matters because each `invoke()` call crosses the IPC bridge and makes an HTTP request — we don't want to repeat that on every page transition.
- **`gcTime: 10 minutes`** — After data becomes stale and the component unmounts, React Query keeps the cached data for 10 more minutes before garbage-collecting it. If the user returns within that window, they see stale data immediately while a background refetch runs.

The provider nesting order matters: `StrictMode` → `QueryClientProvider` → `BrowserRouter` → `App`. React Query needs to be above the router because page components use `useQuery` hooks, and the router needs to be above `App` because `App` defines `<Routes>`.

## Code Listing 17B-5 — Tauri Command Wrappers (`lib/commands.ts`)

This module is the **single point of contact** between the React frontend and the Rust backend. Every Tauri `invoke()` call goes through here.

```typescript
import { invoke } from '@tauri-apps/api/core';

export const commands = {
  // Blockchain
  getBlockchainInfo: () => invoke('get_blockchain_info'),
  getLatestBlocks: () => invoke('get_latest_blocks'),
  getAllBlocks: () => invoke('get_all_blocks'),
  getBlockByHash: (hash: string) => invoke('get_block_by_hash', { hash }),

  // Wallet
  createWallet: (label?: string) => invoke('create_wallet', { label }),
  getWalletInfo: (address: string) =>
    invoke('get_wallet_info', { address }),
  getBalance: (address: string) => invoke('get_balance', { address }),
  sendTransaction: (
    fromAddress: string,
    toAddress: string,
    amount: number
  ) =>
    invoke('send_transaction', {
      fromAddress,
      toAddress,
      amount,
    }),
  getTxHistory: (address: string) =>
    invoke('get_tx_history', { address }),
  getAllAddresses: () => invoke('get_all_addresses'),

  // Transactions
  getMempool: () => invoke('get_mempool'),
  getMempoolTransaction: (txid: string) =>
    invoke('get_mempool_transaction', { txid }),
  getAllTransactions: () => invoke('get_all_transactions'),
  getAddressTransactions: (address: string) =>
    invoke('get_address_transactions', { address }),

  // Mining
  getMiningInfo: () => invoke('get_mining_info'),
  generateBlocks: (address: string, nblocks: number, maxtries?: number) =>
    invoke('generate_blocks', {
      address,
      nblocks,
      maxtries,
    }),

  // Health
  healthCheck: () => invoke('health_check'),
  livenessCheck: () => invoke('liveness_check'),
  readinessCheck: () => invoke('readiness_check'),

  // Settings
  getConfig: () => invoke('get_config'),
  updateConfig: (baseUrl: string, apiKey: string) =>
    invoke('update_config', {
      baseUrl,
      apiKey,
    }),
  checkConnection: () => invoke('check_connection'),
};
```

Each method is a thin wrapper around `invoke()`. The first argument is the command name (must match the Rust function name exactly), and the second is the payload object. Tauri serializes this object to JSON and deserializes it into the Rust function's parameters.

The property names in the payload use **camelCase** (JavaScript convention), and Tauri automatically converts them to **snake_case** (Rust convention) when deserializing. So `{ fromAddress }` in TypeScript becomes `from_address: String` in Rust.

By centralizing all `invoke()` calls here, we get a single place to add logging, modify payload shapes, or swap out the Tauri backend for a mock during testing.

## Code Listing 17B-20 — Utility Functions (`lib/utils.ts`)

Small, pure helper functions used throughout the frontend. These have no Tauri or React dependencies.

```typescript
/**
 * Format satoshis to BTC string
 */
export function formatBTC(satoshis: number): string {
  if (typeof satoshis !== 'number') return '0 BTC';
  const btc = satoshis / 100_000_000;
  return `${btc.toFixed(8)} BTC`;
}

/**
 * Truncate hash with ellipsis
 */
export function truncateHash(hash: string, chars: number = 12): string {
  if (hash.length <= chars) return hash;
  const half = Math.floor(chars / 2);
  return `${hash.slice(0, half)}...${hash.slice(-half)}`;
}

/**
 * Format ISO date string to locale date
 */
export function formatDate(iso: string): string {
  try {
    const date = new Date(iso);
    return date.toLocaleString();
  } catch {
    return iso;
  }
}

/**
 * Merge className strings (like clsx)
 */
export function cn(...classes: (string | undefined | null | false)[]): string {
  return classes.filter(Boolean).join(' ');
}

/**
 * Format large numbers with commas
 */
export function formatNumber(num: number): string {
  return num.toLocaleString();
}

/**
 * Safe JSON stringify with indentation
 */
export function stringifyJSON(obj: any, indent: number = 2): string {
  try {
    return JSON.stringify(obj, null, indent);
  } catch {
    return String(obj);
  }
}

/**
 * Debounce function
 */
export function debounce<T extends (...args: any[]) => any>(
  func: T,
  wait: number
): (...args: Parameters<T>) => void {
  let timeout: NodeJS.Timeout;
  return (...args: Parameters<T>) => {
    clearTimeout(timeout);
    timeout = setTimeout(() => func(...args), wait);
  };
}
```

`formatBTC` divides satoshis by 100,000,000 (1 BTC = 100 million satoshis) and formats to 8 decimal places — the standard Bitcoin display format. `truncateHash` shows the first and last few characters of a block hash or txid with "..." in the middle — useful in tables where a full 64-character hash would be too wide. `cn` is a minimal className merger (like the popular `clsx` library) — we use it throughout the components for conditional CSS classes.

## Code Listing 17B-21 — Global State Store (`store/useAppStore.ts`)

We use Zustand for client-side UI state. This store manages everything that is **not** server data (server data lives in React Query).

```typescript
import { create } from 'zustand';

interface AppStore {
  activeMenu: string;
  expandedSections: string[];
  theme: 'dark' | 'light';
  status: string;
  statusTimestamp: number;
  toggleTheme: () => void;
  setStatus: (message: string) => void;
  toggleSection: (section: string) => void;
  setActiveMenu: (menu: string) => void;
}

export const useAppStore = create<AppStore>((set) => ({
  activeMenu: '/blockchain/info',
  expandedSections: ['blockchain', 'wallet'],
  theme: 'dark',
  status: '',
  statusTimestamp: 0,

  toggleTheme: () =>
    set((state) => {
      const newTheme = state.theme === 'dark' ? 'light' : 'dark';
      if (typeof document !== 'undefined') {
        document.documentElement.classList.toggle('dark', newTheme === 'dark');
        document.documentElement.classList.toggle('light', newTheme === 'light');
      }
      return { theme: newTheme };
    }),

  setStatus: (message: string) =>
    set({
      status: message,
      statusTimestamp: Date.now(),
    }),

  toggleSection: (section: string) =>
    set((state) => ({
      expandedSections: state.expandedSections.includes(section)
        ? state.expandedSections.filter((s) => s !== section)
        : [...state.expandedSections, section],
    })),

  setActiveMenu: (menu: string) => set({ activeMenu: menu }),
}));
```

Zustand's `create` function builds a store with both state and actions in a single call. The store holds:

- **`activeMenu`** — The currently active route path (used to highlight the sidebar item).
- **`expandedSections`** — Which sidebar sections are expanded. Defaults to `['blockchain', 'wallet']` — the two most commonly used sections start open.
- **`theme`** — Dark or light mode. `toggleTheme` also manipulates the DOM directly (`document.documentElement.classList`) because Tailwind's dark mode uses the `dark` class on the HTML element.
- **`status`** / **`statusTimestamp`** — The status bar message shown at the bottom of the window. Pages call `setStatus("Operation completed")` after successful API calls.

Unlike Redux, Zustand does not require reducers, action creators, or dispatch calls. Components access the store with `const { theme, toggleTheme } = useAppStore()`.

## Code Listing 17B-22 — Custom Hook: `useInvoke` (`hooks/useInvoke.ts`)

A generic hook for calling Tauri commands with loading, error, and data state management.

```typescript
import { useState, useCallback } from 'react';
import { invoke } from '@tauri-apps/api/core';

interface UseInvokeOptions {
  onSuccess?: (data: any) => void;
  onError?: (error: string) => void;
}

interface UseInvokeResult<T> {
  data: T | null;
  loading: boolean;
  error: string | null;
  execute: (command: string, payload?: Record<string, any>) => Promise<T | null>;
}

export function useInvoke<T = any>(
  options: UseInvokeOptions = {}
): UseInvokeResult<T> {
  const [data, setData] = useState<T | null>(null);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState<string | null>(null);

  const execute = useCallback(
    async (command: string, payload?: Record<string, any>) => {
      setLoading(true);
      setError(null);

      try {
        const result = await invoke<T>(command, payload);
        setData(result);
        options.onSuccess?.(result);
        return result;
      } catch (err) {
        const errorMessage =
          err instanceof Error ? err.message : String(err);
        setError(errorMessage);
        options.onError?.(errorMessage);
        return null;
      } finally {
        setLoading(false);
      }
    },
    [options]
  );

  return { data, loading, error, execute };
}
```

This hook is used for **imperative** Tauri calls — operations triggered by a button click (like "Create Wallet" or "Send Transaction"). For **declarative** data fetching (like loading blockchain info on page mount), we use React Query's `useQuery` instead.

The hook returns `{ data, loading, error, execute }`. Components call `execute('command_name', { payload })` and the hook manages the lifecycle: set loading → call invoke → set data or error → clear loading. The optional `onSuccess`/`onError` callbacks allow the caller to react to results (e.g., show a toast, navigate to another page).

## Code Listing 17B-23 — Custom Hook: `useClipboard` (`hooks/useClipboard.ts`)

A clipboard hook that wraps Tauri's clipboard plugin and provides user feedback through toast notifications.

```typescript
import { useState, useCallback } from 'react';
import { writeText } from '@tauri-apps/plugin-clipboard-manager';
import { useToast } from '../components/Toast';

export function useClipboard() {
  const { showToast } = useToast();
  const [copied, setCopied] = useState(false);

  const copy = useCallback(
    async (text: string) => {
      try {
        await writeText(text);
        setCopied(true);
        showToast({
          type: 'success',
          message: 'Copied to clipboard',
        });
        setTimeout(() => setCopied(false), 2000);
      } catch (error) {
        const errorMessage =
          error instanceof Error ? error.message : 'Failed to copy';
        showToast({
          type: 'error',
          message: errorMessage,
        });
      }
    },
    [showToast]
  );

  return { copy, copied };
}
```

This hook is used by `DataCard` (copy button on key-value pairs) and `JsonViewer` (copy entire JSON). The `copied` boolean state resets after 2 seconds — useful if you want to show a "Copied!" label that auto-dismisses.

The `writeText` function comes from `@tauri-apps/plugin-clipboard-manager` — it writes to the native system clipboard through Tauri's plugin system, not through the browser's `navigator.clipboard` API (which may be restricted in desktop webviews).

## Code Listing 17B-13 — ConnectionBadge (`components/ConnectionBadge.tsx`)

A real-time connection indicator that polls the Bitcoin API server every 30 seconds.

```tsx
import { useEffect, useState } from 'react';
import { useQuery } from '@tanstack/react-query';
import { commands } from '../lib/commands';

function ConnectionBadge() {
  const [isConnected, setIsConnected] = useState(false);

  const { data } = useQuery({
    queryKey: ['connection'],
    queryFn: async () => {
      try {
        const result = await commands.checkConnection();
        return result;
      } catch {
        return { connected: false };
      }
    },
    refetchInterval: 30000,
    retry: false,
  });

  useEffect(() => {
    if (data && typeof data === 'object' && 'connected' in data) {
      setIsConnected(data.connected as boolean);
    }
  }, [data]);

  return (
    <div className="flex items-center gap-2 rounded-full bg-slate-800 px-15 py-1.5">
      <div
        className={`h-2 w-2 rounded-full ${
          isConnected ? 'bg-green-500' : 'bg-red-500'
        }`}
      />
      <span className="text-sm font-medium">
        {isConnected ? 'Connected' : 'Disconnected'}
      </span>
    </div>
  );
}

export default ConnectionBadge;
```

This component uses React Query's `refetchInterval` to poll the `check_connection` command every 30 seconds. The `retry: false` option prevents React Query from retrying failed connection checks — if the server is down, we want to show "Disconnected" immediately, not after multiple retry attempts.

We explicitly ignore `src-tauri/` from file watching because Rust file changes trigger Tauri's own rebuild, not Vite's.
