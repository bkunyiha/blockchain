## The three page patterns

Every page in the application follows one of these three patterns:

### Pattern 1: Auto-fetch display page (uses `useQuery`)

These pages load data automatically on mount and optionally re-poll at a fixed interval. They display results using `DataCard`, `DataTable`, or `JsonViewer`. Examples: BlockchainInfoPage, LatestBlocksPage, AllBlocksPage, MempoolPage, AllTransactionsPage, AddressListPage, MiningInfoPage, HealthPage.

```text
Component mounts → useQuery fires → invoke() → Rust command → HTTP → data
                                                                    ↓
                 ←── React Query caches ←── IPC response ←── JSON result
                                    ↓
              re-render with DataCard / DataTable / JsonViewer
```

### Pattern 2: Search page (uses `useInvoke` hook)

These pages have a text input and a search button. The user enters a value (hash, address, txid), clicks "Search" or presses Enter, and the result is displayed. Examples: BlockByHashPage, WalletInfoPage, BalancePage, TxHistoryPage, MempoolTxPage, AddressTxPage.

```text
User types value → clicks Search → useInvoke.execute() → invoke() → Rust → HTTP
                                                                          ↓
                              ←── loading/error/data state updates ←── result
                                                ↓
                          re-render with JsonViewer / DataCard
```

### Pattern 3: Form submission page (uses `react-hook-form` + `zod` + `useInvoke`)

These pages have a multi-field form with validation. The user fills in the form, submits, and optionally confirms the action. Examples: CreateWalletPage, SendBitcoinPage, GenerateBlocksPage.

```text
User fills form → submit → zod validates → (optional ConfirmDialog) → useInvoke.execute()
                                                                           ↓
                                    ←── success/error display ←── Rust → HTTP result
```

## Blockchain Pages

### Listing 17C-1 — BlockchainInfoPage (Pattern 1: auto-fetch display)

This is the default landing page. It fetches blockchain statistics and displays them as key-value cards.

```tsx
import { useQuery } from '@tanstack/react-query';
import { commands } from '../../lib/commands';
import DataCard, { CardItem } from '../../components/DataCard';
import { BlockchainInfo } from '../../types';
import { useAppStore } from '../../store/useAppStore';

function BlockchainInfoPage() {
  const setStatus = useAppStore((state) => state.setStatus);

  const { data, isLoading, error } = useQuery({
    queryKey: ['blockchainInfo'],
    queryFn: async () => {
      const result = await commands.getBlockchainInfo();
      return result as BlockchainInfo;
    },
    refetchInterval: 30000,
    onSuccess: () => {
      setStatus('Blockchain info fetched successfully');
    },
    onError: (err) => {
      setStatus(`Error: ${err instanceof Error ? err.message : 'Unknown error'}`);
    },
  });

  if (isLoading) {
    return (
      <div className="flex items-center justify-center p-8">
        <div className="text-slate-400">Loading blockchain info...</div>
      </div>
    );
  }

  if (error || !data) {
    return (
      <div className="rounded-lg border border-red-700 bg-red-900/20 p-22 text-red-200">
        <p>Failed to load blockchain info</p>
        {error && <p className="text-sm opacity-75">{String(error)}</p>}
      </div>
    );
  }

  const items: CardItem[] = [
    {
      label: 'Current Height',
      value: data.height,
      format: 'number',
      copyable: true,
    },
    {
      label: 'Total Blocks',
      value: data.total_blocks,
      format: 'number',
      copyable: true,
    },
    {
      label: 'Difficulty',
      value: data.difficulty,
      format: 'number',
      copyable: true,
    },
    {
      label: 'Best Block Hash',
      value: data.best_block_hash,
      format: 'hash',
      copyable: true,
    },
  ];

  return <DataCard title="Blockchain Information" items={items} />;
}

export default BlockchainInfoPage;
```

This page demonstrates the Pattern 1 structure: `useQuery` with `refetchInterval: 30000` re-polls the blockchain info every 30 seconds. The `queryKey: ['blockchainInfo']` uniquely identifies this cached data in React Query. The `onSuccess`/`onError` callbacks update the status bar (via Zustand) so the user sees feedback at the bottom of the window.

The `CardItem[]` array configures the `DataCard` component: each item specifies a `label`, `value`, optional `format` (number, hash, btc, date), and whether the value is `copyable` (shows a copy button).

### Listing 17C-2 — LatestBlocksPage (Pattern 1: table display)

```tsx
import { useQuery } from '@tanstack/react-query';
import { commands } from '../../lib/commands';
import DataTable, { Column } from '../../components/DataTable';
import { BlockSummary } from '../../types';

  ];

  if (isLoading) {

    <div className="space-y-6">

    </div>
  );

export default LatestBlocksPage;
```

This page uses `DataTable` instead of `DataCard` because block data is naturally tabular. The `columns` array defines how each field renders: the `hash` column uses `truncateHash` to show a shortened version, and the `time` column converts a Unix timestamp (seconds since epoch) to a human-readable date by multiplying by 1000 (JavaScript `Date` expects milliseconds).

### Listing 17C-4 — BlockByHashPage (Pattern 2: search)

```tsx
import { useState } from 'react';
import { Search } from 'lucide-react';
import { useInvoke } from '../../hooks/useInvoke';
import { commands } from '../../lib/commands';
import JsonViewer from '../../components/JsonViewer';
import { useAppStore } from '../../store/useAppStore';

function BlockByHashPage() {
  const [hash, setHash] = useState('');
  const setStatus = useAppStore((state) => state.setStatus);

  const { data, loading, error, execute } = useInvoke({
    onSuccess: () => {
      setStatus('Block fetched successfully');
    },
    onError: (err) => {
      setStatus(`Error: ${err}`);
    },
  });

  const handleSearch = async () => {
    if (!hash.trim()) {
      setStatus('Please enter a block hash');
      return;
    }
    await execute('get_block_by_hash', { hash: hash.trim() });
  };

  const handleKeyDown = (e: React.KeyboardEvent) => {
    if (e.key === 'Enter') {
      handleSearch();
    }
  };

  return (
    <div className="space-y-6">
      <h1 className="text-2xl font-bold">Block by Hash</h1>

      <div className="rounded-lg border border-slate-700 bg-slate-800 p-6">
        <label className="block text-sm font-medium mb-2">Block Hash</label>
        <div className="flex gap-2">
          <input
            type="text"
            value={hash}
            onChange={(e) => setHash(e.target.value)}
            onKeyDown={handleKeyDown}
            placeholder="Enter block hash..."
            className="flex-1 rounded-lg border border-slate-600 bg-slate-700 px-4 py-2 text-slate-100 placeholder-slate-400 focus:border-yellow-500 focus:outline-none"
          />
          <button
            onClick={handleSearch}
            disabled={loading}
            className="flex items-center gap-2 rounded-lg bg-yellow-500 px-20 py-2 font-medium text-slate-900 hover:bg-yellow-600 disabled:opacity-50"
          >
            <Search className="h-5 w-5" />
            {loading ? 'Searching...' : 'Search'}
          </button>
        </div>
      </div>

      {error && (
        <div className="rounded-lg border border-red-700 bg-red-900/20 p-22 text-red-200">
          <p>Failed to fetch block</p>
          <p className="text-sm opacity-75">{error}</p>
        </div>
      )}

      {data && (
        <JsonViewer data={data} title="Block Details" maxHeight={700} />
      )}
    </div>
  );
}

export default BlockByHashPage;
```

This page demonstrates Pattern 2: `useInvoke` instead of `useQuery`. The data is not fetched on mount — the user must enter a hash and click "Search" (or press Enter). The `handleKeyDown` listener on the input field triggers the search on Enter. Notice that `useInvoke` uses the command name string (`'get_block_by_hash'`) rather than the typed wrapper from `commands.ts` — this is a design choice that keeps `useInvoke` generic.

## Wallet Pages

### Listing 17C-5 — SendBitcoinPage (Pattern 3: form with confirmation)

This is the most complex page — it combines `react-hook-form` + `zod` validation + a confirmation dialog.

```tsx
import { useState } from 'react';
import { useForm } from 'react-hook-form';
import { zodResolver } from '@hookform/resolvers/zod';
import { z } from 'zod';
import { Send } from 'lucide-react';
import { useInvoke } from '../../hooks/useInvoke';
import { commands } from '../../lib/commands';
import ConfirmDialog from '../../components/ConfirmDialog';
import { useAppStore } from '../../store/useAppStore';

const sendSchema = z.object({
  fromAddress: z.string().min(1, 'From address is required'),
  toAddress: z.string().min(1, 'To address is required'),
  amount: z.coerce.number().positive('Amount must be positive'),
});

type SendForm = z.infer<typeof sendSchema>;

interface SendResponse {
  txid: string;
}

function SendBitcoinPage() {
  const [confirmOpen, setConfirmOpen] = useState(false);
  const [formData, setFormData] = useState<SendForm | null>(null);
  const setStatus = useAppStore((state) => state.setStatus);

  const { register, handleSubmit, formState: { errors }, reset } = useForm<SendForm>({
    resolver: zodResolver(sendSchema),
  });

  const { data, loading, error, execute } = useInvoke<SendResponse>({
    onSuccess: () => {
      setStatus('Transaction sent successfully');
      reset();
      setFormData(null);
    },
    onError: (err) => {
      setStatus(`Error: ${err}`);
    },
  });

  const onSubmit = (data: SendForm) => {
    setFormData(data);
    setConfirmOpen(true);
  };

  const handleConfirm = async () => {
    if (!formData) return;
    setConfirmOpen(false);
    await execute('send_transaction', {
      fromAddress: formData.fromAddress,
      toAddress: formData.toAddress,
      amount: formData.amount,
    });
  };

  return (
    <div className="space-y-6">
      <h1 className="text-2xl font-bold">Send Bitcoin</h1>

      <form onSubmit={handleSubmit(onSubmit)} className="space-y-6">
        <div className="rounded-lg border border-slate-700 bg-slate-800 p-6">
          <div className="space-y-4">
            <div>
              <label className="block text-sm font-medium mb-2">
                From Address
              </label>
              <input
                type="text"
                {...register('fromAddress')}
                placeholder="Bitcoin address to send from..."
                className="w-full rounded-lg border border-slate-600 bg-slate-700 px-4 py-2 text-slate-100 placeholder-slate-400 focus:border-yellow-500 focus:outline-none"
              />
              {errors.fromAddress && (
                <p className="mt-1 text-sm text-red-400">
                  {errors.fromAddress.message}
                </p>
              )}
            </div>

            <div>
              <label className="block text-sm font-medium mb-2">
                To Address
              </label>
              <input
                type="text"
                {...register('toAddress')}
                placeholder="Bitcoin address to send to..."
                className="w-full rounded-lg border border-slate-600 bg-slate-700 px-4 py-2 text-slate-100 placeholder-slate-400 focus:border-yellow-500 focus:outline-none"
              />
              {errors.toAddress && (
                <p className="mt-1 text-sm text-red-400">
                  {errors.toAddress.message}
                </p>
              )}
            </div>

            <div>
              <label className="block text-sm font-medium mb-2">
                Amount (Satoshis)
              </label>
              <input
                type="number"
                {...register('amount')}
                placeholder="0"
                className="w-full rounded-lg border border-slate-600 bg-slate-700 px-4 py-2 text-slate-100 placeholder-slate-400 focus:border-yellow-500 focus:outline-none"
              />
              {errors.amount && (
                <p className="mt-1 text-sm text-red-400">
                  {errors.amount.message}
                </p>
              )}
            </div>

            <button
              type="submit"
              disabled={loading}
              className="flex items-center gap-2 rounded-lg bg-yellow-500 px-20 py-2 font-medium text-slate-900 hover:bg-yellow-600 disabled:opacity-50"
            >
              <Send className="h-5 w-5" />
              {loading ? 'Sending...' : 'Send'}
            </button>
          </div>
        </div>
      </form>

      {error && (
        <div className="rounded-lg border border-red-700 bg-red-900/20 p-22 text-red-200">
          <p>Failed to send transaction</p>
          <p className="text-sm opacity-75">{error}</p>
        </div>
      )}

      {data && (
        <div className="rounded-lg border border-green-700 bg-green-900/20 p-20 text-green-200">
          <h3 className="mb-4 font-bold">Transaction Sent Successfully!</h3>
          <div className="space-y-2">
            <div>
              <p className="text-sm font-medium">TXID:</p>
              <code className="block break-all rounded-lg bg-slate-800 p-2 font-mono text-sm text-slate-100 mt-1">
                {data.txid}
              </code>
            </div>
          </div>
        </div>
      )}

      {formData && (
        <ConfirmDialog
          open={confirmOpen}
          onConfirm={handleConfirm}
          onCancel={() => {
            setConfirmOpen(false);
            setFormData(null);
          }}
          title="Confirm Transaction"
          description={`Send ${formData.amount} satoshis from ${formData.fromAddress} to ${formData.toAddress}?`}
          confirmLabel="Send"
          variant="default"
        />
      )}
    </div>
  );
}

export default SendBitcoinPage;
```

The flow is: user fills the form → `handleSubmit` triggers zod validation → if valid, `onSubmit` saves the form data and opens the `ConfirmDialog` → user clicks "Send" in the dialog → `handleConfirm` closes the dialog and calls `execute('send_transaction', ...)` → on success, the form resets and the TXID is displayed in green.

The zod schema validates:
- `fromAddress`: required non-empty string
- `toAddress`: required non-empty string
- `amount`: `z.coerce.number().positive()` — the `coerce` prefix converts the string from the HTML input to a number before validation

`react-hook-form`'s `register()` function connects each input field to the form state. `{...register('fromAddress')}` spreads the necessary `onChange`, `onBlur`, `ref`, and `name` props onto the input element. Validation errors show below each field as red text.

### Listing 17C-20 — CreateWalletPage (Pattern 3: simple form)

              onKeyDown={handleKeyDown}

              className="w-full rounded-lg border border-slate-600 bg-slate-700 px-4 py-2 text-slate-100 placeholder-slate-400 focus:border-yellow-500 focus:outline-none"
            />
          </div>

          <button
            onClick={handleCreate}
            disabled={loading}
            className="flex items-center gap-2 rounded-lg bg-yellow-500 px-20 py-2 font-medium text-slate-900 hover:bg-yellow-600 disabled:opacity-50"
          >
            <Plus className="h-5 w-5" />
            {loading ? 'Creating...' : 'Create Wallet'}
          </button>
        </div>
      </div>

      {error && (
        <div className="rounded-lg border border-red-700 bg-red-900/20 p-22 text-red-200">
          <p>Failed to create wallet</p>
          <p className="text-sm opacity-75">{error}</p>
        </div>
      )}

      {data && (
        <div className="rounded-lg border border-green-700 bg-green-900/20 p-20 text-green-200">

          <div className="space-y-3">
            <div>
              <p className="text-sm font-medium mb-2">Address:</p>
              <div className="flex items-center gap-2 rounded-lg bg-slate-800 p-3">

This is a simpler Pattern 3 variant — no zod schema needed because the only field is optional. The success state shows the created wallet address with a copy button (using the `useClipboard` hook). `label || undefined` converts an empty string to `undefined` so the Rust backend receives `None` for the `Option<String>` parameter.

## Health Page

### Listing 17C-22 — HealthPage (Pattern 1: multi-query display)

The health page is unique because it runs three independent queries simultaneously.

```tsx
import { useEffect, useState } from 'react';
import { useQuery } from '@tanstack/react-query';
import { RotateCcw, CheckCircle, XCircle } from 'lucide-react';
import { commands } from '../../lib/commands';
import { useAppStore } from '../../store/useAppStore';

interface HealthStatus {
  status: string;
  ok?: boolean;
  healthy?: boolean;
  ready?: boolean;
}

function HealthPage() {
  const setStatus = useAppStore((state) => state.setStatus);
  const [refetchInterval, setRefetchInterval] = useState(60000);

  const health = useQuery({
    queryKey: ['healthCheck'],
    queryFn: async () => {
      const result = await commands.healthCheck();
      return result as HealthStatus;
    },
    refetchInterval,
  });

  const liveness = useQuery({
    queryKey: ['livenessCheck'],
    queryFn: async () => {
      const result = await commands.livenessCheck();
      return result as HealthStatus;
    },
    refetchInterval,
  });

  const readiness = useQuery({
    queryKey: ['readinessCheck'],
    queryFn: async () => {
      const result = await commands.readinessCheck();
      return result as HealthStatus;
    },
    refetchInterval,
  });

  useEffect(() => {
    setStatus('Health checks completed');
  }, [health.data, liveness.data, readiness.data, setStatus]);

  const handleRefresh = () => {
    health.refetch();
    liveness.refetch();
    readiness.refetch();
  };

  const renderCheck = (
    title: string,
    data: HealthStatus | undefined,
    isLoading: boolean,
    error: Error | null
  ) => {
    let isHealthy = false;
    if (data) {
      isHealthy = !!(data.healthy || data.ok || data.ready || data.status === 'ok');
    }

    return (
      <div className="rounded-lg border border-slate-700 bg-slate-800 p-6">
        <div className="flex items-center gap-4">
          {isLoading ? (
            <div className="h-12 w-12 rounded-full bg-slate-700 flex items-center justify-center animate-spin">
              <RotateCcw className="h-20 w-20 text-slate-400" />
            </div>
          ) : error ? (
            <div className="h-12 w-12 rounded-full bg-red-900 flex items-center justify-center">
              <XCircle className="h-20 w-20 text-red-400" />
            </div>
          ) : isHealthy ? (
            <div className="h-12 w-12 rounded-full bg-green-900 flex items-center justify-center">
              <CheckCircle className="h-20 w-20 text-green-400" />
            </div>
          ) : (
            <div className="h-12 w-12 rounded-full bg-red-900 flex items-center justify-center">
              <XCircle className="h-20 w-20 text-red-400" />
            </div>
          )}

          <div className="flex-1">
            <h3 className="text-lg font-bold">{title}</h3>
            <p className="text-sm text-slate-400 mt-1">
              {isLoading ? 'Checking...' : error ? 'Failed to check' : isHealthy ? 'Healthy' : 'Unhealthy'}
            </p>
          </div>

          <button
            onClick={handleRefresh}
            className="rounded-lg bg-slate-700 p-2 hover:bg-slate-600 transition-colors"
            title="Refresh all checks"
          >
            <RotateCcw className="h-5 w-5" />
          </button>
        </div>
      </div>
    );
  };

  return (
    <div className="space-y-6">
      <div className="flex items-center justify-between">
        <h1 className="text-2xl font-bold">Health Status</h1>
        <button
          onClick={handleRefresh}
          className="flex items-center gap-2 rounded-lg bg-yellow-500 px-4 py-2 font-medium text-slate-900 hover:bg-yellow-600 transition-colors"
        >
          <RotateCcw className="h-5 w-5" />
          Refresh All
        </button>
      </div>

      <div className="grid gap-4 sm:grid-cols-1 lg:grid-cols-3">
        {renderCheck('Health Check', health.data, health.isLoading, health.error as Error | null)}
        {renderCheck('Liveness Check', liveness.data, liveness.isLoading, liveness.error as Error | null)}
        {renderCheck('Readiness Check', readiness.data, readiness.isLoading, readiness.error as Error | null)}
      </div>

      <div className="rounded-lg border border-slate-700 bg-slate-800 p-6">
        <h3 className="mb-4 font-bold">Information</h3>
        <ul className="space-y-2 text-sm text-slate-400">
          <li><strong>Health Check:</strong> Verifies basic service availability</li>
          <li><strong>Liveness Check:</strong> Confirms the service is running</li>
          <li><strong>Readiness Check:</strong> Ensures the service is ready to serve requests</li>
          <li><strong>Auto-refresh:</strong> Checks are automatically refreshed every 60 seconds</li>
        </ul>
      </div>
    </div>
  );
}

export default HealthPage;
```

This page uses three separate `useQuery` hooks — each running on its own cache key and refetch interval. The `renderCheck` helper function extracts the visual pattern (spinning loader / green checkmark / red X) so we don't repeat it three times. The `isHealthy` check is flexible — it checks multiple possible response shapes (`data.healthy`, `data.ok`, `data.ready`, `data.status === 'ok'`) because the API server may return different health response formats.

## Mining Page

### Listing 17C-23 — GenerateBlocksPage (Pattern 3: form with zod)

```tsx
import { useState } from 'react';
import { useForm } from 'react-hook-form';
import { zodResolver } from '@hookform/resolvers/zod';
import { z } from 'zod';
import { Zap } from 'lucide-react';
import { useInvoke } from '../../hooks/useInvoke';
import { commands } from '../../lib/commands';
import JsonViewer from '../../components/JsonViewer';
import { useAppStore } from '../../store/useAppStore';

const generateSchema = z.object({
  address: z.string().min(1, 'Address is required'),
  nblocks: z.coerce.number().positive('Number of blocks must be positive').integer(),
  maxtries: z.coerce.number().positive().integer().optional(),
});

type GenerateForm = z.infer<typeof generateSchema>;

function GenerateBlocksPage() {
  const setStatus = useAppStore((state) => state.setStatus);
  const { register, handleSubmit, formState: { errors } } = useForm<GenerateForm>({
    resolver: zodResolver(generateSchema),
  });

  const { data, loading, error, execute } = useInvoke({
    onSuccess: () => {
      setStatus('Blocks generated successfully');
    },
    onError: (err) => {
      setStatus(`Error: ${err}`);
    },
  });

  const onSubmit = async (formData: GenerateForm) => {
    await execute('generate_blocks', {
      address: formData.address,
      nblocks: formData.nblocks,
      maxtries: formData.maxtries,
    });
  };

  return (
    <div className="space-y-6">
      <h1 className="text-2xl font-bold">Generate Blocks</h1>

      <form onSubmit={handleSubmit(onSubmit)} className="space-y-6">
        <div className="rounded-lg border border-slate-700 bg-slate-800 p-6">
          <div className="space-y-4">
            <div>
              <label className="block text-sm font-medium mb-2">Miner Address</label>
              <input
                type="text"
                {...register('address')}
                placeholder="Bitcoin address for mining rewards..."
                className="w-full rounded-lg border border-slate-600 bg-slate-700 px-4 py-2 text-slate-100 placeholder-slate-400 focus:border-yellow-500 focus:outline-none"
              />
              {errors.address && <p className="mt-1 text-sm text-red-400">{errors.address.message}</p>}
            </div>

            <div>
              <label className="block text-sm font-medium mb-2">Number of Blocks</label>
              <input
                type="number"
                {...register('nblocks')}
                placeholder="1"
                className="w-full rounded-lg border border-slate-600 bg-slate-700 px-4 py-2 text-slate-100 placeholder-slate-400 focus:border-yellow-500 focus:outline-none"
              />
              {errors.nblocks && <p className="mt-1 text-sm text-red-400">{errors.nblocks.message}</p>}
            </div>

            <div>
              <label className="block text-sm font-medium mb-2">Max Tries (Optional)</label>
              <input
                type="number"
                {...register('maxtries')}
                placeholder="1000000"
                className="w-full rounded-lg border border-slate-600 bg-slate-700 px-4 py-2 text-slate-100 placeholder-slate-400 focus:border-yellow-500 focus:outline-none"
              />
              {errors.maxtries && <p className="mt-1 text-sm text-red-400">{errors.maxtries.message}</p>}
            </div>

            <button
              type="submit"
              disabled={loading}
              className="flex items-center gap-2 rounded-lg bg-yellow-500 px-20 py-2 font-medium text-slate-900 hover:bg-yellow-600 disabled:opacity-50"
            >
              <Zap className="h-5 w-5" />
              {loading ? 'Generating...' : 'Generate'}
            </button>
          </div>
        </div>
      </form>

      {error && (
        <div className="rounded-lg border border-red-700 bg-red-900/20 p-22 text-red-200">
          <p>Failed to generate blocks</p>
          <p className="text-sm opacity-75">{error}</p>
        </div>
      )}

      {data && <JsonViewer data={data} title="Generation Results" maxHeight={700} />}
    </div>
  );
}

export default GenerateBlocksPage;
```

Unlike SendBitcoinPage, this form submits immediately without a confirmation dialog — block generation on a development node is a non-destructive operation. The `maxtries` field is optional in both the zod schema (`.optional()`) and the Rust backend (`Option<u32>`).
